System Architecture

How the authentication pipeline is built
🔑

Password Security

Passwords never stored in plaintext. bcryptjs hashes with 12 salt rounds — computationally expensive to brute force. Each hash is unique even for identical passwords.

bcryptjs 12 salt rounds compare()
🎫

JWT Token Strategy

Dual-token: short-lived access token (15 min) in memory; long-lived refresh token (7 days) in an httpOnly cookie. Prevents XSS token theft while enabling persistent sessions.

jsonwebtoken httpOnly cookie RS256
🛡

Role-Based Access Control

Three-tier role system: admin, moderator, user. Middleware factory requireRole(...roles) guards routes. Roles embedded in JWT payload for stateless checks.

admin moderator user requireRole()
🔄

Session Management

Silent token refresh via /api/auth/refresh every 14 minutes. On mount, app restores session from cookie. Logout clears both cookie and in-memory token atomically.

cookie-parser auto-refresh sameSite strict
